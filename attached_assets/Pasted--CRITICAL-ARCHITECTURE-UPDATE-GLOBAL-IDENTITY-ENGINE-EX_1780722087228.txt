Role: Lead Cloud Architect & Flutter/Supabase Engineer.
Context: We are upgrading the app to a robust Multi-Tenant SaaS platform. Users must be able to use a single "Stride" QR Code across multiple independent associations without breaking database isolation. We also need a massive data import module.

STEP 1: GLOBAL IDENTITY & TENANT-LINKING DATABASE ARCHITECTURE
- Refactor the database logic to support a "Global User Wallet".
- Table 1: `global_users` (Stores universal data: UserID, Global_QR_Code, First Name, Last Name, Email).
- Table 2: `tenant_memberships` (Junction table linking global UserID to a specific TenantID/Association, with a 'status' like 'invited', 'active', 'expired').

STEP 2: THE BULK DATA IMPORT MODULE (EXCEL/CSV PARSER)
- Create a new UI section in the Standard Admin dashboard: "Import Members Data".
- Implement a robust file picker allowing the Admin to upload `.csv`, `.xls`, or `.xlsx` files.

- Insert this parsed data into a `pending_invites` table linked to the Admin's TenantID, mapping out exactly how many total members and dependants are expected to join.
